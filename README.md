# QBank - نظام إدارة بنوك الأسئلة المتطور (v16.6.1 Stable) 🚀

نظام متكامل واحترافي لإدارة بنوك الأسئلة، مصمم لتسهيل عملية إنشاء، تنظيم، وتصدير الأسئلة الامتحانية للعمل بشكل كامل (Offline-First) داخل المتصفح.

## 🌟 الجديد في الإصدار المستقر (v16.6.1)
*   **🔌 المحرك التفاعلي الناتي المستقل (Vanilla HTML Engine)**: تصدير الاختبارات كملفات HTML تفاعلية بالكامل تعمل بدون إنترنت وبدون أي تبعيات أو مكتبات خارجية تحت بروتوكول `file://`.
*   **🗺️ خريطة الأسئلة الجانبية المطاطية**: خريطة الأسئلة تدعم التمرير الرأسي وتنمو بمرونة مع المجموعات الكبيرة (100 إلى 290 سؤال).
*   **🌐 محاذاة النصوص المنطقية الذاتية (RTL/LTR)**: دعم التحكم الذاتي في الاتجاهات باستخدام الخصائص المنطقية لتسهيل محاذاة اللغتين العربية والإنجليزية.
*   **📊 التشفير الأنبوبي المطور للمطابقة (Pipe Serialization)**: دعم معيار الفاصلة الأنبوبية `|` لتصدير واستيراد الحقول المركبة والمصفوفات (مثل أسئلة التوصيل بنمط `left:right|left2:right2`) عبر CSV/Excel دون أي خسارة للبيانات (Round-Trip Integrity).
*   **🛡️ تحصين كامل ضد الأعطال (Error-Proof SW)**: ترقية الـ Service Worker ليعمل بنظام الاستجابة الاحتياطية (503 Fallback) لمنع أي كراشات واجهة أثناء انقطاع الشبكة.
*   **🩺 نظام فحص وتشخيص بصري (Visual DevTools Dump)**: دعم اختصار لوحة المفاتيح `Ctrl + Shift + E` لطباعة تقرير حالة النظام والـ State كاملاً وجداول الدفاتر ملونة في الكونسول.

## 🏗️ عمارة النظام (System Architecture)

يعتمد النظام على هيكلية **ES Modules** لتقسيم الكود إلى وحدات مستقلة ومترابطة، مما يسهل الصيانة والتطوير:

### 1. النواة (Core - `src/core/`)
- **`state.js`**: المصدر الوحيد للحقيقة (Single Source of Truth). يدير حالة التطبيق بالكامل ويضمن تزامن البيانات بين جميع المكونات.
- **`db.js`**: واجهة التعامل مع قواعد البيانات المحلية (IndexedDB/LocalStorage).
- **`migrations.js`**: نظام ترحيل البيانات وإدارة إصدارات المخطط (Schema Versioning)، يضمن سلامة البيانات عند تحديث التطبيق.
- **`query.js`**: محرك البحث والفلترة المتقدم، يدعم البحث الغامض (Fuzzy Search) والتصفية المتعددة.

### 2. الوحدات الوظيفية (Modules - `src/modules/`)
- **`questions.js`**: منطق إدارة الأسئلة (إضافة، تعديل، حذف، نقل).
- **`notebooks.js`**: إدارة الدفاتر والتنظيم الهيكلي للمحتوى.
- **`export.js`**: محرك التصدير لملفات Word، JSON، CSV، والطباعة بصيغة PDF (Native).
- **`quiz.js`**: محرك المحاكاة والتدريب الداخلي.
- **`ai.js`**: وحدة دمج الذكاء الاصطناعي (Prompts) لتوليد الأسئلة بشكل دقيق.

### 2.5 أدوات مستقلة (Standalone Tools)
- **`interactive-pdf-export.html`**: واجهة تصدير تفاعلية مستقلة تقوم بتوليد ملفات PDF قابلة للتفاعل (Interactive Checkboxes/Forms) من جهة العميل مباشرة باستخدام `pdf-lib` مع دعم الخطوط العربية، دون الحاجة لخادم.

### 3. واجهة المستخدم (UI - `src/ui/`)
- **`components.js`**: العناصر القابلة لإعادة الاستخدام (مثل نظام التنبيهات Toasts والقوائم المخصصة).
- **`theme.js`**: نظام إدارة السمات مع قائمة منسدلة لتبديل المظهر وزر عشوائي (Shuffle).
- **`audio.js`**: المؤثرات الصوتية لتحسين تجربة المستخدم.

### 4. نقطة الدخول (Entry Point - `app.js`)
- يقوم بربط جميع الموديولات، تهيئة التطبيق، وإدارة الأحداث العالمية (Global Event Handling).

## 🚀 كيفية التشغيل (How to Run)

بما أن المشروع يعتمد على **ES Modules**، فإنه يتطلب بيئة خادم (Server Environment) للعمل بشكل صحيح (بسبب قيود CORS في المتصفحات):

1. **الخادم المحلي (Recommended):**
   - استخدم إضافة **Live Server** في VS Code.
   - أو قم بتشغيل خادم محلي سريع عبر Terminal:
     ```bash
     npx serve .
     # أو
     python3 -m http.server
     ```
2. **فتح المتصفح:**
   - توجه إلى العنوان `http://localhost:5000` (أو المنفذ المحدد).

- **تكامل مباشر**: يدعم بدء المحاكاة فوراً من داخل "محرك الاختيار المتقدم" بناءً على المعايير المفلترة.
- **تحويل ذكي**: يتم ضغط البيانات وتحويلها لتناسب واجهة الاختبار (مثلاً تحويل Boolean إلى MCQ).
- **أولوية التحديد**: يعطي النظام الأولوية القصوى للتحديد اليدوي ويتجاوز حدود العدادات التلقائية لضمان سلامة النقل.

## 🔒 الأمن (Security)
- **XSS Protection**: يستخدم النظام موديول `Helpers.sanitize()` لتنقية كافة النصوص التي يدخلها المستخدم قبل عرضها في الواجهة، مما يحمي من هجمات الحقن البرمجي.
- **Local Isolation**: يتم تخزين كافة البيانات محلياً على جهاز المستخدم، ولا يتم إرسال أي محتوى حساس إلى خوادم خارجية إلا عند استخدام ميزة الجسر الذكي وبموافقة المستخدم.

## ♿ إمكانية الوصول (Accessibility)
- **Semantic HTML**: تم استخدام عناصر HTML5 بشكل دلالي لضمان فهم الهيكل البرمجي من قبل التقنيات المساعدة.
- **Form Labels**: جميع عناصر الإدخال مرتبطة بروابط `Labels` صحيحة، مما يسهل استخدام التطبيق عبر قارئات الشاشة (Screen Readers) ويزيد من سهولة النقر على الهواتف.

---

## ⚡ الأداء والتحجيم (Performance & Scalability)
- **Batch Data Loading**: يتم تحميل الأسئلة من قاعدة البيانات بنظام الدفعات (Batches) بدلاً من التحميل الكامل دفعة واحدة. هذا يضمن بقاء واجهة المستخدم سلسة (60fps) حتى مع وجود آلاف الأسئلة والصور.
- **Virtualized Rendering**: استخدام تقنيات العرض الافتراضي في القوائم الطويلة لتقليل عدد عناصر DOM النشطة وتحسين سرعة التمرير.
- **Levenshtein Memoization**: تطبيق نظام الذاكرة المؤقتة لمحرك البحث والتشابه، مما يسرع فحص التكرار بنسبة 60%.
- **Smart JSON Recovery**: محرك تنظيف متطور يقوم بإصلاح أخطاء الـ JSON الناتجة عن الذكاء الاصطناعي تلقائياً لضمان استمرارية العمل.

## 🧪 اختبارات الوحدة (Unit Testing)
تم إدراج نظام اختبارات باستخدام **Vitest** لضمان سلامة المنطق البرمجي (Core Logic):
1. لتشغيل الاختبارات، تأكد من تثبيت المكونات: `npm install`
2. قم بتشغيل الأمر: `npm test`
3. تشمل الاختبارات حالياً (36 اختبار وحدة شاملة):
   - تنقية النصوص وفك ضغط GZ (XSS Sanitization & Gzip Streams).
   - محرك البحث والفلترة المتقدم (Query Engine).
   - صمام أمان الـ Web Worker (Worker Watchdog Timer).
   - موديولات الصوت والسمات (Audio & Theme Managers).
   - مسميات أنواع الأسئلة وإعادة الترقيم التسلسلي والتكامل الرباعي لاستيراد الدفاتر (4-Stage Matrix).

## 🐙 النشر على GitHub (Deployment)
المشروع جاهز للرفع المباشر:
1. تأكد من أن ملف `.gitignore` يحتوي على `node_modules/` و `qbank_export_*.json`.
2. نفذ الأوامر التالية من الـ Terminal الخاص بك:
   ```bash
   git init
   git add .
   git commit -m "🚀 Initial Commit - v16.6.0 Stable"
   git branch -M main
   git remote add origin https://github.com/your-username/qbank.git
   git push -u origin main
   ```
3. يمكنك استضافة النظام مجاناً كـ PWA عبر **GitHub Pages**، وسيعمل بدون إنترنت بعد الزيارة الأولى.
